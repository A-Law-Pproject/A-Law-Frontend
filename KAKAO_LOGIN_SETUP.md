# 카카오 로그인 설정 가이드

## 1. 카카오 개발자 앱 생성

1. [카카오 개발자 사이트](https://developers.kakao.com/)에 접속합니다.
2. 로그인 후 **내 애플리케이션**으로 이동합니다.
3. **애플리케이션 추가하기** 버튼을 클릭합니다.
4. 앱 이름(예: A-Law), 사업자명을 입력하고 저장합니다.

## 2. 플랫폼 등록

1. 생성한 앱을 선택합니다.
2. **플랫폼** 메뉴로 이동합니다.
3. **Web 플랫폼 등록** 버튼을 클릭합니다.
4. 사이트 도메인을 입력합니다:
   - 개발 환경: `http://localhost:5173` (Vite 기본 포트)
   - 운영 환경: 실제 서비스 도메인

## 3. Redirect URI 설정

1. **카카오 로그인** 메뉴로 이동합니다.
2. **활성화 설정**을 ON으로 변경합니다.
3. **Redirect URI** 등록:
   - 개발 환경: `http://localhost:5173`
   - 운영 환경: 실제 서비스 도메인

## 4. JavaScript 키 복사

1. **앱 설정** > **앱 키** 메뉴로 이동합니다.
2. **JavaScript 키**를 복사합니다.

## 5. 환경변수 설정

1. 프로젝트 루트에 `.env` 파일을 생성합니다 (이미 생성되어 있음).
2. 복사한 JavaScript 키를 입력합니다:

```env
VITE_KAKAO_APP_KEY=your_javascript_key_here
```

## 6. 동의 항목 설정 (선택사항)

1. **카카오 로그인** > **동의 항목** 메뉴로 이동합니다.
2. 필요한 정보 수집 항목을 설정합니다:
   - **닉네임**: 필수 동의
   - **프로필 사진**: 선택 동의
   - **카카오계정(이메일)**: 선택 동의

## 7. 테스트

1. 개발 서버를 재시작합니다:
   ```bash
   npm run dev
   ```

2. 브라우저에서 `http://localhost:5173/mypage`로 이동합니다.

3. **카카오톡 로그인** 버튼을 클릭하여 로그인을 테스트합니다.

## 주의사항

- `.env` 파일은 Git에 커밋하지 마세요 (이미 .gitignore에 추가되어 있음).
- JavaScript 키는 클라이언트에서 사용하므로 보안에 주의하세요.
- 운영 환경 배포 시 환경변수를 별도로 설정해야 합니다.

## 문제 해결

### "Kakao SDK not loaded" 오류
- 브라우저 콘솔에서 카카오 SDK가 로드되었는지 확인하세요.
- `index.html`에 SDK 스크립트가 정상적으로 추가되었는지 확인하세요.

### 로그인 팝업이 뜨지 않음
- 브라우저의 팝업 차단 설정을 확인하세요.
- 카카오 개발자 사이트에서 플랫폼과 Redirect URI가 정확히 등록되었는지 확인하세요.

### 로그인 후 정보가 저장되지 않음
- 브라우저의 localStorage 사용이 허용되어 있는지 확인하세요.
- 시크릿 모드에서는 localStorage가 제한될 수 있습니다.
